# i18n

Translations live in `src/locales/{de,en}/{namespace}.json`. Default language: DE, fallback: EN.

## Namespaces

| Namespace | File | Contains |
|---|---|---|
| `common` | `common.json` | Nav, actions, shared UI labels, toasts, profile strings, smart split labels |
| `screens` | `screens.json` | Screen-specific strings (titles, subtitles, field labels) |
| `errors` | `errors.json` | Validation and error messages |

## Usage

```tsx
const { t } = useTranslation(['common', 'screens', 'errors']);

t('common:actions.save')           // namespace:key
t('screens:setup.title')
t('common:toast.shiftSaved')
t('screens:setup.employeeCount_one', { count: 1 })  // with interpolation
```

## Adding a String

1. Add to `src/locales/de/{namespace}.json`
2. Add to `src/locales/en/{namespace}.json`
3. Reference with `t('namespace:key')`

Both files must stay in sync. Always add to both languages.

## Plurals

Use `_one` / `_other` suffixes:

```json
"employeeCount_one": "{{count}} Mitarbeiter",
"employeeCount_other": "{{count}} Mitarbeiter"
```

```tsx
t('screens:setup.employeeCount_other', { count: n })
```
